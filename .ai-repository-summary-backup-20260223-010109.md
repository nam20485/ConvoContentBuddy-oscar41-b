# ConvoContentBuddy Repository Summary

> AI-powered background listening tool for real-time programming interview assistance.

## High-Level Overview

**ConvoContentBuddy** is a highly resilient, AI-powered application that processes live speech audio, identifies algorithmic problems being discussed, and retrieves optimal solutions via the Gemini API—all without requiring manual user input.

| Attribute | Value |
|-----------|-------|
| **Repository Type** | .NET 10 Solution (Aspire-orchestrated microservices) |
| **Languages** | C# 14, JavaScript (Interop), SQL |
| **Frameworks** | .NET 10, ASP.NET Core 10, Blazor WebAssembly 10, .NET Aspire 10 |
| **Target Runtimes** | .NET 10.0 (browser, linux, windows) |
| **Status** | ✅ Solution Structure Created |

## Architecture Highlights

### Technology Stack
- **AI/LLM:** Microsoft.SemanticKernel, Microsoft.Extensions.AI, Gemini 2.5 Flash
- **Data & State:** Qdrant (vector DB), PostgreSQL (pgvector), Redis (SignalR backplane)
- **Resilience:** Polly policies, Triple Modular Redundancy (TMR)
- **Real-time:** SignalR with Redis backplane for WebSocket failover
- **Styling:** Tailwind CSS

### Key Architectural Patterns
1. **Triple Modular Redundancy (TMR):** 3 replicated API instances for aerospace-grade resilience
2. **Hybrid Vector-Graph Search:** Qdrant semantic search + PostgreSQL graph traversal
3. **Zero-Interaction UI:** Blazor WASM with autonomous SignalR-driven updates
4. **Safe Mode Fallback:** Local-only operation when cloud services are unreachable

## Development Workflow

This repository uses **dynamic workflows** orchestrated via the `AGENTS.md` entry point. Workflow assignments are fetched from the remote canonical repository (`nam20485/agent-instructions`).

### Branch Resolution
- Configured branch in `AGENTS.md` determines instruction source
- Default: `main` branch
